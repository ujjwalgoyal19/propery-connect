## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/ujjwalgoyal19/propery-connect.git
    cd propery-connect
    ```
  </Step>

<Step title="Install dependencies">
  This command installs dependencies for all workspaces in the monorepo: ```bash npm install ```
</Step>

  <Step title="Set up environment variables">
    Create environment files for both frontend and backend:

    **Backend (.env in apps/api/)**
    ```bash
    PORT=3005
    NODE_ENV=development

    # Supabase
    SUPABASE_URL=your_supabase_url
    SUPABASE_SERVICE_ROLE_KEY=your_service_role_key

    # Twilio
    TWILIO_ACCOUNT_SID=your_account_sid
    TWILIO_AUTH_TOKEN=your_auth_token
    TWILIO_WHATSAPP_NUMBER=your_whatsapp_number

    # Razorpay
    RAZORPAY_KEY_ID=your_key_id
    RAZORPAY_KEY_SECRET=your_key_secret

    # Other
    FRONTEND_URL=http://localhost:3000
    JWT_SECRET=your_jwt_secret
    ```

    **Frontend (.env.local in apps/web/)**
    ```bash
    NEXT_PUBLIC_API_URL=http://localhost:3005/api
    NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
    NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
    ```

  </Step>

  <Step title="Run database migrations">
    Apply the initial schema to your Supabase database:
    ```bash
    # Using Supabase CLI
    npx supabase db push
    ```

    Or manually run the SQL in `supabase/migrations/00001_initial_schema.sql` in your Supabase SQL editor.

  </Step>

  <Step title="Start the development servers">
    Start all services simultaneously:
    ```bash
    npm run dev
    ```

    Or start individual services:
    ```bash
    # Frontend only (runs on port 3000)
    npm run dev --workspace=@propery-connect/web

    # Backend only (runs on port 3005)
    npm run dev --workspace=@propery-connect/api

    # Documentation only
    npm run dev --workspace=@propery-connect/docs
    ```

  </Step>
</Steps>

## Common Issues

<Warning>
  **Port conflicts**: If port 3000 is already in use, Next.js will automatically use port 3002. Make
  sure to update your `NEXT_PUBLIC_API_URL` accordingly.
</Warning>

<AccordionGroup>
  <Accordion icon="triangle-exclamation" title="Environment validation errors">
    If you see environment validation errors on backend startup, ensure all required variables are set in `apps/api/.env`. The backend uses Zod to validate environment variables and will exit if any are missing.
  </Accordion>

<Accordion icon="triangle-exclamation" title="Database connection issues">
  Verify your Supabase credentials are correct and that your IP is allowed in Supabase's database
  settings.
</Accordion>

  <Accordion icon="triangle-exclamation" title="Module not found errors">
    If you see module resolution errors, try:
    ```bash
    # Clear node_modules and reinstall
    rm -rf node_modules
    rm package-lock.json
    npm install
    ```
  </Accordion>
</AccordionGroup>
